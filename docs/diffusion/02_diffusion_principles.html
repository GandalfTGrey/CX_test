03｜核心直觉

Diffusion 的训练可以看成“人为破坏数据，然后训练模型学会修复”。

一句话：Forward process 把真实数据逐步加噪变成高斯噪声；Reverse process 学习从噪声一步步去噪回真实数据分布。
阶段做什么谁决定学习了吗
Forward / 加噪$x_0 \rightarrow x_t$固定公式和 noise schedule不学习
Reverse / 去噪$x_t \rightarrow x_{t-1}$神经网络 $p_\theta$ / $\epsilon_\theta$学习
Training预测噪声或 clean dataMSE / ELBO / score matching学习 denoising direction
Sampling从 $x_T \sim \mathcal{N}(0,I)$ 生成 $x_0$反向采样算法使用训练好的模型

Knowledge Point 1

05｜Forward Process：加噪过程

一句话解释

Forward process 是一个固定的马尔可夫加噪过程，把真实数据 $x_0$ 逐步变成接近标准高斯噪声的 $x_T$。

解决什么问题

如果我们能把任意数据逐步破坏成标准高斯噪声，那么生成时就可以反过来：从标准高斯噪声逐步恢复真实数据。

数学公式

$$q(x_t \mid x_{t-1}) = \mathcal{N}(x_t;\sqrt{1-\beta_t}x_{t-1}, \beta_t I)$$

利用高斯分布的组合性质，可以直接从 $x_0$ 采样任意时刻的 $x_t$：

$$q(x_t \mid x_0)=\mathcal{N}(x_t;\sqrt{\bar\alpha_t}x_0,(1-\bar\alpha_t)I)$$
$$x_t = \sqrt{\bar\alpha_t}x_0 + \sqrt{1-\bar\alpha_t}\epsilon,\quad \epsilon \sim \mathcal{N}(0,I)$$

公式-代码对应

# Formula:
# x_t = sqrt(alpha_bar_t) * x_0 + sqrt(1 - alpha_bar_t) * epsilon

def add_noise(x0, t, alpha_bars):
    noise = torch.randn_like(x0)                         # epsilon ~ N(0, I)
    sqrt_ab = alpha_bars[t].sqrt().view(-1, 1, 1)         # sqrt(alpha_bar_t)
    sqrt_1mab = (1 - alpha_bars[t]).sqrt().view(-1, 1, 1) # sqrt(1-alpha_bar_t)
    x_t = sqrt_ab * x0 + sqrt_1mab * noise                # noisy sample
    return x_t, noise
Forward process 不需要训练，也不是模型学出来的。它是人为规定的噪声破坏过程。
Knowledge Point 2

06｜Noise Schedule：噪声调度

Noise schedule 决定每个 timestep 加多少噪声，也就是 $\beta_t$ 如何随 $t$ 变化。

$$\alpha_t = 1-\beta_t,\qquad \bar{\alpha}_t=\prod_{s=1}^{t}\alpha_s$$
def make_linear_schedule(num_steps, beta_start=1e-4, beta_end=0.02):
    betas = torch.linspace(beta_start, beta_end, num_steps)
    alphas = 1.0 - betas
    alpha_bars = torch.cumprod(alphas, dim=0)
    return betas, alphas, alpha_bars

Noise schedule 不是单纯工程细节。它会影响训练难度、采样质量、不同 timestep 的信噪比和模型要学习的 denoising 任务分布。

Knowledge Point 3

07｜Reverse Process：去噪过程

Reverse process 学习从 $x_t$ 预测 $x_{t-1}$，逐步把噪声样本恢复成真实数据。

$$p_\theta(x_{t-1}\mid x_t)=\mathcal{N}(x_{t-1};\mu_\theta(x_t,t),\Sigma_\theta(x_t,t))$$

Forward process 是“把清晰图像/轨迹弄脏”；Reverse process 是训练一个清洁工，每次只清理一点点噪声。

class TinyDenoiser(nn.Module):
    def __init__(self, x_dim, hidden_dim=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(x_dim + 1, hidden_dim),
            nn.SiLU(),
            nn.Linear(hidden_dim, x_dim),
        )

    def forward(self, x_t, t):
        inp = torch.cat([x_t, t], dim=-1)
        pred_noise = self.net(inp)
        return pred_noise
Knowledge Point 4

08｜DDPM 训练目标：预测噪声

DDPM 常用的简化训练目标是让模型预测 forward process 中加入的噪声 $\epsilon$。

$$\mathcal{L}_{simple}=\mathbb{E}_{x_0,t,\epsilon}\left[\left\|\epsilon-\epsilon_\theta(x_t,t)\right\|_2^2\right]$$

如果模型能准确预测 $x_t$ 里有多少噪声，就可以把这些噪声从 $x_t$ 中扣掉，从而估计 clean data 或上一步 $x_{t-1}$。

x_t, noise = add_noise(x0, t, alpha_bars)      # create noisy sample
pred_noise = model(x_t, t_embed)               # epsilon_theta(x_t, t)
loss = F.mse_loss(pred_noise, noise)           # noise prediction loss
Knowledge Point 5

09｜Score Matching 视角

Score 是 log-density 对输入的梯度，表示“当前样本往哪里移动，概率密度会上升最快”。

$$s_t(x_t)=\nabla_{x_t}\log p_t(x_t)$$

把数据分布想象成地形。高概率区域是山峰，低概率区域是山谷。Score 就是当前位置的上坡方向。去噪过程就是沿着更高概率的数据区域移动。

$$s_\theta(x_t,t)\approx -\frac{\epsilon_\theta(x_t,t)}{\sqrt{1-\bar\alpha_t}}$$
pred_noise = model(x_t, t_embed)
sqrt_1mab = (1 - alpha_bars[t]).sqrt().view(-1, 1)
score = -pred_noise / sqrt_1mab
Score 不是一个分类分数，也不是 reward。它是数据分布的 log-density 梯度。

10｜ε / x0 / v 预测参数化

参数化模型预测什么优点代码变量
$\epsilon$-prediction预测加入的噪声DDPM 常见形式，稳定、直观pred_noise
$x_0$-prediction预测 clean data直观，可直接得到去噪样本pred_x0
$v$-prediction预测 signal 和 noise 的组合变量常用于改进不同 SNR 下的训练稳定性pred_v
$$\hat{x}_0=\frac{x_t-\sqrt{1-\bar\alpha_t}\epsilon_\theta(x_t,t)}{\sqrt{\bar\alpha_t}}$$
def predict_x0_from_noise(x_t, t, pred_noise, alpha_bars):
    sqrt_ab = alpha_bars[t].sqrt().view(-1, 1, 1)
    sqrt_1mab = (1 - alpha_bars[t]).sqrt().view(-1, 1, 1)
    pred_x0 = (x_t - sqrt_1mab * pred_noise) / sqrt_ab
    return pred_x0
Knowledge Point 6

11｜Sampling：从噪声生成数据

Sampling 是从 $x_T \sim \mathcal{N}(0,I)$ 出发，反复调用 denoiser，逐步得到 $x_0$。

@torch.no_grad()
def ddpm_sample(model, shape, alpha_bars, num_steps):
    x = torch.randn(shape)  # x_T ~ N(0, I)
    for t in reversed(range(num_steps)):
        t_tensor = torch.full((shape[0],), t, dtype=torch.long)
        pred_noise = model(x, t_tensor)
        pred_x0 = predict_x0_from_noise(x, t_tensor, pred_noise, alpha_bars)
        x = one_step_denoise(x, pred_noise, t_tensor)  # conceptual placeholder
    return x

12｜条件生成与 Guidance

无条件 diffusion 学习 $p(x)$；条件 diffusion 学习 $p(x|c)$。条件 $c$ 可以是类别、文本、图像、轨迹历史、地图、场景 token 或 reward。

$$\epsilon_\theta(x_t,t,c)$$
pred_noise = model(x_t, t, cond)  # cond may be text, class, map, scene tokens, etc.

Classifier-Free Guidance 直觉

训练时随机丢掉 condition，让模型同时学会 conditional 和 unconditional denoising。推理时放大两者差异，让生成结果更贴合 condition。

$$\epsilon_{cfg} = \epsilon_\theta(x_t,t,\varnothing) + w\left(\epsilon_\theta(x_t,t,c)-\epsilon_\theta(x_t,t,\varnothing)\right)$$
eps_uncond = model(x_t, t, cond=None)
eps_cond = model(x_t, t, cond=cond)
eps_cfg = eps_uncond + guidance_scale * (eps_cond - eps_uncond)

13｜Latent Diffusion

Latent Diffusion 不在原始数据空间里扩散，而是在 autoencoder 的 latent space 里扩散，从而降低计算成本。

$$z_0 = E(x_0), \quad z_t = \sqrt{\bar\alpha_t}z_0 + \sqrt{1-\bar\alpha_t}\epsilon$$
z0 = encoder(x0)
z_t, noise = add_noise(z0, t, alpha_bars)
pred_noise = denoiser(z_t, t, cond)
loss = F.mse_loss(pred_noise, noise)

在自动驾驶中，类似思想可以迁移为：不直接在 raw sensor 或 full trajectory 空间扩散，而是在 compact scene latent / trajectory latent 中扩散。

14｜迁移到 Trajectory Diffusion

把 diffusion 从图像迁移到轨迹时，核心变量 $x_0$ 不再是图像，而是 future trajectory 或 action sequence。

$$x_0 = Y = [(x_1,y_1),(x_2,y_2),...,(x_T,y_T)]$$
$$x_t = \sqrt{\bar\alpha_t}Y + \sqrt{1-\bar\alpha_t}\epsilon$$

图像 diffusion

$x_0$ 是像素矩阵；condition 可以是 text prompt；输出质量看视觉质量。

轨迹 diffusion

$x_0$ 是未来轨迹；condition 是场景、地图、历史运动；输出质量看安全、舒适、规则、闭环表现。

这为 2.3 自动驾驶应用章节打基础：Diffusion Planner 本质上就是 scene-conditioned trajectory denoising。

15｜最小 PyTorch 代码骨架

import torch
import torch.nn as nn
import torch.nn.functional as F

def make_schedule(num_steps=1000, beta_start=1e-4, beta_end=0.02):
    betas = torch.linspace(beta_start, beta_end, num_steps)
    alphas = 1.0 - betas
    alpha_bars = torch.cumprod(alphas, dim=0)
    return betas, alphas, alpha_bars

def extract(a, t, x_shape):
    out = a.gather(0, t).float()
    return out.view(t.shape[0], *([1] * (len(x_shape) - 1)))

def add_noise(x0, t, alpha_bars):
    noise = torch.randn_like(x0)
    sqrt_ab = extract(alpha_bars.sqrt(), t, x0.shape)
    sqrt_1mab = extract((1 - alpha_bars).sqrt(), t, x0.shape)
    x_t = sqrt_ab * x0 + sqrt_1mab * noise
    return x_t, noise

class MLPDenoiser(nn.Module):
    def __init__(self, x_dim, hidden=256):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(x_dim + 1, hidden), nn.SiLU(),
            nn.Linear(hidden, hidden), nn.SiLU(),
            nn.Linear(hidden, x_dim),
        )
    def forward(self, x_t, t):
        t = t.float().unsqueeze(-1) / 1000.0
        inp = torch.cat([x_t, t], dim=-1)
        return self.net(inp)

B, x_dim, num_steps = 32, 2, 1000
betas, alphas, alpha_bars = make_schedule(num_steps)
model = MLPDenoiser(x_dim)
x0 = torch.randn(B, x_dim)
t = torch.randint(0, num_steps, (B,))
x_t, noise = add_noise(x0, t, alpha_bars)
pred_noise = model(x_t, t)
loss = F.mse_loss(pred_noise, noise)
loss.backward()

16｜常见误解

误解 1：Forward process 是模型学出来的

不是。Forward process 是固定加噪公式，模型学的是 reverse process。

误解 2：Diffusion 只能生成图片

不是。只要数据可以表示为向量、序列或 latent，就可以建模，例如 action sequence 和 trajectory。

误解 3：Loss 只是普通 MSE，没有特殊意义

虽然形式是 MSE，但预测噪声等价于学习去噪方向，与 score matching 有联系。

误解 4：采样步数越多一定越好

更多步通常更慢，质量也受 schedule、模型、任务和 guidance 影响。实际工程需要质量-速度权衡。
